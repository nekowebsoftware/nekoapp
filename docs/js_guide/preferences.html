Application preferences

Application preferences are application stylesheets, elements, element classes and events that are used by your application. All we need is to define them.

Application stylesheets

Application stylesheets needed to be defined to optimize page load by including them after application initialization proccess has started. So we do like this:

var your_app = new nekoapp({
     ...
     applicationStylesheets : {
          stylesheet_name : "URL to your stylesheet"
     }
     ...
});

By default "default", "colors" and "ui" stylesheets are needed.

  • your_app.main.css (default stylesheet) - used for creating app layout
  • your_app.colors.css (colors stylesheet) - used for color vaiables (both non-dark and dark mode)
  • your_app.ui.css (user interface stylesheet) - used for user interface of your app

Application classes

Next is appication CSS classes that can be used by your application. All you need is define required ones:

var your_app = new nekoapp({
     ...
     applicationClasses : {
          class_name : "Your CSS class name"
     }
     ...
});

This is a list of classes names that are needed and used by framework:

  • UIButtonBackground - button background part
  • UIButtonContent - button content part
  • UIDefaultButton - for default button type
  • UIColorButton - for color button type
  • UIGlassButton - for glass button type
  • UIPaneButton - for pane button type
  • UICircleButton - for circle or icon-type graphic button type
  • UICheckBoxIcon - for checkbox icon
  • UICheckBoxContent - for checkbox content or label
  • UIRadioButtonIcon - for radio button icon
  • UIRadioButtonContent - for radio button content or label
  • UITextBoxInput - for textbox input part
  • UITextBoxStroke - for textbox stroke part (stroked textbox type)
  • UIStrokedTextBox - for stroked textbox type
  • UISimpleTextBox - for simple textbox type
  • UIHighlightedTextBox - for highlighted textbox type
  • UIComboBoxValue - for combobox value part
  • UIComboBoxIcon - for combobox icon-type graphic part
  • UIComboBoxOptions - for combobox options
  • headerContainer - for header container
  • headerNavigationItem - for header navigation item
  • localeChangeList - for language list in loacle change window
  • progressBarInner - for inner part of progress bar that above

Application elements

Since the feature as Custom Elements (or also called as Web Components) were introduced and given opportunity to create custom elements for specified purposes. So also Neko Web Application aims to use that feature and make it easy to use just because good applications needs their own components. All we need is define elements and their powers (prototype functions or maybe variables/constants) for our application:

var your_app = new nekoapp({
     ...
     applicationElements : {
          element_name : {
               tag : "Your element tag name" ,
               prototype : {
                    // All prototype stuff belongs here
               }
          }
     }
     ...
});

Remember that all custom elements are defined by specifying dash "-" symbol in tag name

And also this is list of elements that are needed and used by framework

  • localizedStringElement - for localized strings
  • graphicElement - for graphic elements
  • graphicsLibraryElement - for graphic library
  • graphicsSetElement - for graphic set
  • animationElement - for animations
  • animationSpriteElement - for animation sprites
  • UIElement - for user interface elements
  • buttonElement - for buttons
  • checkBoxElement - for checkboxes
  • radioButtonElement - for radio buttons
  • textBoxElement - for textboxes
  • comboboxElement - for comboboxes
  • tooltipElement - for tooltips
  • moduleElement - for application modules
  • headerElement - for header module
  • footerElement - for footer module
  • windowElement - for application windows
  • windowTitle - for window title
  • overlayElement - for overlay
  • headerLogoElement - for header logo
  • headerNavigationElement - for header navigation
  • headerNavigationItem - for header navigation item
  • spinnerElement - for spinner
  • loadScreenElement - for loading screen
  • progressBarElement - for progress bar

Application events

Neko Web Application Events is a complex of event listeners attached to applcation object (window object also affectable). Some events are registering in init process, but some needs to be registered manually if they needed to use.

var your_app = new nekoapp({
     ...
     applicationEvents : {
          event_name : {
               target : object_name or "string addressing to future object" ,
               event_listener : function(){ /* something will happen */ }
          }
     }
     ...
});

--- or ---

var your_app = new nekoapp({
     ...
     applicationEvents : {
          event_name : new nekoapp.event({
               target : object_name or "string addressing to future object" ,
               event_listener : function(){ /* something will happen */ }
          })
     }
     ...
});

This feature is incomplete and not everything work as should yet, so use standard event listeners for being sure.

Neko Web Application object listens to these event listeners:

  • oninit - triggers when application is finishing its initialization to do some stuff before app is ready to use.
  • More listeners will appear in time...

Application classes and events can be declared outside application constructor.

But some window and nekoapp events needs to be declared before initialization because Neko Web Application registers them automatically during initialization process.
